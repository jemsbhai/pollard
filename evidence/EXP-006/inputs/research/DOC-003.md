# DOC-003: Replay, verification, and seals

Record mode executes a function and stores its result. Hybrid mode reuses a
matching stored node and otherwise records a new one. Replay mode never invokes
the supplied function; a missing recording fails. Before a stored result is
served, Pollard verifies the node identity and ancestry.

`pollard verify` checks stored identities and result digests. A subtree seal is
a rolling SHA-256 report over deterministic traversal order, node IDs, and
result digests. Mutable metadata is excluded from the seal. The ledger is
tamper-evident rather than tamper-proof: verification can detect changed
history, but it cannot prevent deletion of the complete store and seal.
