# DOC-001: Identity and canonical serialization

Pollard records each semantic step as a node whose identity commits to the
parent node, step kind, attempt number, and identity payload. The identity
document is serialized as sorted, compact UTF-8 JSON beneath the `pollard/v1`
domain before SHA-256 hashing. Results and mutable metadata are not identity
inputs. Identity payloads accept null, strings, booleans, integers, lists, and
string-keyed objects; floats and bytes are rejected.

This separation lets two calls beneath the same parent reuse one stored result
when their inputs match, while different parents or retry attempts remain
distinct. A changed payload produces a different node even if the provider
would return identical text.
