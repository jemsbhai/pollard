# DOC-002: Budgets and concurrent reservations

Pollard checks known charges before executing a step. Exact step and request
meters can refuse a call before a configured shared limit is crossed. Token
charges are often estimates: a transactional store reserves the estimate,
executes the call, then settles the actual charge. The documented concurrent
bound is the limit plus positive actual-minus-estimate errors from calls that
were already admitted.

SQLite coordinates writers on one host. PostgreSQL provides the shared
transactional arbiter for several processes or hosts using the same logical
store. Expiring leases return active capacity after a worker disappears, but
they cannot undo provider spend that occurred before failure.
